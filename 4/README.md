## Исключения и обработка ошибок в Java

---

## Описание

Данная лабораторная работа посвящена изучению механизма обработки исключений (exception handling) в Java. В ходе работы реализованы три программы, демонстрирующие практическое применение конструкций try-catch-finally, создание пользовательских исключений и логирование ошибок. Работа направлена на развитие навыков написания надёжного и отказоустойчивого кода, способного корректно обрабатывать различные ошибочные ситуации.

---

## Ключевые концепции

### Исключения (Exceptions)

Исключение — это объект, который описывает ошибку или нестандартную ситуацию во время выполнения программы. Вместо того чтобы просто завершить программу с ошибкой, исключение позволяет передать управление специальному обработчику и корректно выйти из неправильной ситуации.

### Блоки try-catch-finally

Структура обработки исключений состоит из трёх частей:
- **try** — блок с опасным кодом, который может выбросить исключение
- **catch** — блок обработки конкретного типа исключения
- **finally** — блок, который выполняется всегда, независимо от наличия ошибки

### Типы исключений

**Checked-исключения** (проверяемые):
- Компилятор заставляет либо перехватить, либо объявить через `throws`
- Примеры: `IOException`, `FileNotFoundException`

**Unchecked-исключения** (непроверяемые):
- Компилятор не требует обязательно их обрабатывать
- Примеры: `NumberFormatException`, `ArrayIndexOutOfBoundsException`, `NullPointerException`

### Создание пользовательских исключений

Пользовательские исключения создаются наследованием от класса `Exception`:
```java
public class CustomDivisionException extends Exception {
    public CustomDivisionException(String message) {
        super(message);
    }
}
```

### Выброс исключений

Ключевое слово `throw` используется для явного выброса исключения:
```java
if (divisor == 0) {
    throw new CustomDivisionException("Делить на ноль нельзя.");
}
```

### **ArrayAverage.java — Вычисление среднего арифметического массива**

Программа считывает размер массива и его элементы с консоли, вычисляет среднее арифметическое и обрабатывает две группы ошибок.

**Основная логика**:
- Использует `Scanner` для ввода данных с клавиатуры
- Метод `Integer.parseInt()` преобразует строку в целое число
- Метод `split("\\s+")` разбивает строку по пробельным символам

**Обработка ошибок**:
- `NumberFormatException` — обрабатывает ввод нечисловых значений
- `ArrayIndexOutOfBoundsException` — обрабатывает случай, когда введено меньше элементов, чем указано в размере
- `finally` — гарантирует закрытие `Scanner` в любом случае

**Пример ввода/вывода**:

### **CustomDivisionException.java — Пользовательское исключение**

Класс, представляющий собственное исключение для операции деления.

**Структура**:
```java
public class CustomDivisionException extends Exception {
    public CustomDivisionException(String message) {
        super(message);  // Передаёт сообщение об ошибке в родительский класс
    }
}
```

**Особенности**:
- Наследуется от `Exception`, что делает его checked-исключением
- Конструктор принимает строковое сообщение об ошибке
- Может быть отловлено в catch-блоке и использовано в логировании

### **DivisionProgram.java — Программа деления с логированием**

Программа делит два введённых числа, выбрасывает пользовательское исключение при делении на ноль и логирует все ошибки в файл.

**Метод divide()**:
```java
public static double divide(double a, double b) throws CustomDivisionException {
    if (b == 0) {
        throw new CustomDivisionException("Делить на ноль нельзя.");
    }
    return a / b;
}
```
- Объявляет через `throws`, что может выбросить `CustomDivisionException`
- Явно проверяет делитель перед операцией
- Выбрасывает исключение, если делитель равен нулю

**Метод log()**:
```java
public static void log(String text) {
    try (FileWriter writer = new FileWriter("exceptions_log.txt", true)) {
        writer.write(text + System.lineSeparator());
    } catch (IOException e) {
        System.out.println("Ошибка записи в файл.");
    }
}
```
- Использует `try-with-resources` для автоматического закрытия `FileWriter`
- Второй параметр `true` включает режим дозаписи в конец файла
- Перехватывает `IOException` при ошибке записи

**Обработка исключений в main()**:
- `CustomDivisionException` — логирует деление на ноль
- `NumberFormatException` — логирует ввод нечисловых данных
- `finally` — закрывает `Scanner`

### **FileCopy.java — Копирование содержимого файла**

Программа копирует содержимое одного текстового файла в другой с обработкой ошибок открытия и закрытия файлов.

**Основная логика**:
```java
FileReader reader = new FileReader("source.txt");
FileWriter writer = new FileWriter("destination.txt");

int c;
while ((c = reader.read()) != -1) {
    writer.write(c);  // Копирование символ за символом
}
```

**Обработка ошибок открытия**:
- `FileNotFoundException` — возникает, если исходный файл не существует или недоступен

**Обработка ошибок закрытия**:
- `finally` блок содержит вложенные try-catch для закрытия каждого файла отдельно
- Гарантирует, что оба файла будут закрыты даже при ошибке

**Структура finally**:
```java
finally {
    try {
        if (reader != null) {
            reader.close();
        }
    } catch (IOException e) {
        System.out.println("Ошибка при закрытии входного файла.");
    }
    
    try {
        if (writer != null) {
            writer.close();
        }
    } catch (IOException e) {
        System.out.println("Ошибка при закрытии выходного файла.");
    }
}
```

## Содержание заданий

### Задание 1: Вычисление среднего арифметического массива

**Описание**: Программа находит среднее арифметическое элементов массива и обрабатывает ошибки ввода.

**Реализованный функционал**:
- Ввод размера массива через консоль
- Ввод элементов массива одной строкой через пробел
- Вычисление суммы элементов в цикле
- Вычисление среднего арифметического с приведением к `double`
- Вывод результата с адекватной точностью

**Обработка ошибок**:

1. **NumberFormatException** — когда пользователь вводит нечисловое значение
   - Возникает в `Integer.parseInt(scanner.nextLine())`
   - Также возникает в `Integer.parseInt(values[i])`
   - Сообщение: "Ошибка: введено не число."

2. **ArrayIndexOutOfBoundsException** — когда введено меньше элементов, чем указано в размере
   - Возникает при обращении к `values[i]`, который не существует
   - Сообщение: "Ошибка: введено меньше элементов, чем нужно."

**Алгоритм**:
```java
// 1. Ввести размер
int size = Integer.parseInt(scanner.nextLine());
int[] arr = new int[size];

// 2. Разбить строку ввода на элементы
String[] values = scanner.nextLine().trim().split("\\s+");

// 3. Преобразовать строки в числа и вычислить сумму
int sum = 0;
for (int i = 0; i < size; i++) {
    arr[i] = Integer.parseInt(values[i]);
    sum += arr[i];
}

// 4. Вычислить среднее
System.out.println("Среднее: " + (double) sum / size);
```

**Пример работы**:
```
Введите размер массива: 4
Введите элементы массива через пробел:
10 20 30 40
Среднее арифметическое: 25.0
```

**Пример ошибки (нечисловой ввод)**:
```
Введите размер массива: abc
Ошибка: введено не число.
```

**Пример ошибки (недостаточно элементов)**:
```
Введите размер массива: 5
Введите элементы массива через пробел:
1 2 3
Ошибка: введено меньше элементов, чем нужно.
```

### Задание 2: Копирование файла с обработкой ошибок (Вариант 1)

**Описание**: Программа копирует содержимое одного файла в другой, обрабатывая ошибки открытия и закрытия файлов.

**Реализованный функционал**:
- Открытие исходного файла для чтения
- Открытие целевого файла для записи
- Построчное копирование содержимого
- Гарантированное закрытие обоих файлов
- Обработка ошибок на всех этапах

**Обработка ошибок открытия**:

1. **FileNotFoundException** — когда исходный файл не найден
   - Возникает при `new FileReader("source.txt")`
   - Сообщение: "Ошибка: файл не найден."

2. **IOException** — другие ошибки ввода-вывода
   - Возникает при чтении, записи или закрытии
   - Сообщение: "Ошибка при работе с файлом."

**Обработка ошибок закрытия**:
- Каждый файл закрывается в отдельном try-catch блоке
- Гарантирует, что ошибка при закрытии одного файла не помешает закрытию другого
- Специфичные сообщения для каждого файла

**Алгоритм**:
```java
FileReader reader = null;
FileWriter writer = null;

try {
    // 1. Открыть файлы
    reader = new FileReader("source.txt");
    writer = new FileWriter("destination.txt");
    
    // 2. Копировать символ за символом
    int c;
    while ((c = reader.read()) != -1) {
        writer.write(c);
    }
    
    System.out.println("Файл скопирован.");
    
} catch (FileNotFoundException e) {
    System.out.println("Ошибка: файл не найден.");
} catch (IOException e) {
    System.out.println("Ошибка при работе с файлом.");
} finally {
    // 3. Закрыть файлы с обработкой ошибок
    try {
        if (reader != null) reader.close();
    } catch (IOException e) {
        System.out.println("Ошибка при закрытии входного файла.");
    }
    
    try {
        if (writer != null) writer.close();
    } catch (IOException e) {
        System.out.println("Ошибка при закрытии выходного файла.");
    }
}
```

**Сценарий успешного выполнения**:
```
Файл скопирован.
```

**Сценарий ошибки (файл не найден)**:
```
Ошибка: файл не найден.
```

**Сценарий ошибки (нет прав на запись)**:
```
Ошибка при работе с файлом.
```

### Задание 3: Деление с пользовательским исключением (Вариант 1)

**Описание**: Программа делит два числа, выбрасывает пользовательское исключение при делении на ноль и логирует все исключения в файл.

**Реализованный функционал**:

1. **Класс CustomDivisionException**:
   - Пользовательское checked-исключение
   - Наследуется от Exception
   - Содержит конструктор с сообщением об ошибке

2. **Метод divide()**:
   - Проверяет, что делитель не равен нулю
   - Выбрасывает CustomDivisionException при делении на ноль
   - Возвращает результат деления в противном случае

3. **Метод log()**:
   - Логирует информацию об исключении в файл `exceptions_log.txt`
   - Использует режим дозаписи (`true` в FileWriter)
   - Добавляет переход на новую строку для каждой записи

4. **Метод main()**:
   - Вводит два числа с клавиатуры
   - Вызывает метод divide()
   - Обрабатывает CustomDivisionException и NumberFormatException
   - Логирует каждое возникшее исключение

**Обработка исключений**:

1. **CustomDivisionException** — деление на ноль
   - Сообщение об ошибке: "Делить на ноль нельзя."
   - Логируется как: "CustomDivisionException: Делить на ноль нельзя."

2. **NumberFormatException** — ввод нечисловых данных
   - Сообщение об ошибке: "Ошибка: введено не число."
   - Логируется как: "NumberFormatException: введено не число."

**Структура логирования**:
```java
public static void log(String text) {
    try (FileWriter writer = new FileWriter("exceptions_log.txt", true)) {
        writer.write(text + System.lineSeparator());
    } catch (IOException e) {
        System.out.println("Ошибка записи в файл.");
    }
}
```

**Алгоритм**:
```java
try {
    System.out.print("Введите первое число: ");
    double a = Double.parseDouble(scanner.nextLine());
    
    System.out.print("Введите второе число: ");
    double b = Double.parseDouble(scanner.nextLine());

} catch (CustomDivisionException e) {
    System.out.println("Ошибка: " + e.getMessage());
    log(e.getClass().getSimpleName() + ": " + e.getMessage());
} catch (NumberFormatException e) {
    System.out.println("Ошибка: введено не число.");
    log(e.getClass().getSimpleName() + ": введено не число.");
} finally {
    scanner.close();
}
```


## Важные замечания

### Checked vs Unchecked исключения

- **Checked** (IOException, FileNotFoundException) — компилятор заставляет их обрабатывать
- **Unchecked** (NumberFormatException, ArrayIndexOutOfBoundsException) — компилятор не требует, но хорошо их обрабатывать

### Блок finally

- Выполняется всегда, независимо от наличия исключения
- Правильное место для освобождения ресурсов
- Используется даже если в catch выполняется return

### Порядок catch блоков

- Более узкие типы исключений должны идти раньше
- Например, FileNotFoundException раньше IOException
- После того как нужное исключение поймано, остальные catch блоки пропускаются

### Try-with-resources

- В Java 7+ можно использовать `try (ресурс)` для автоматического закрытия
- Ресурс должен реализовать интерфейс AutoCloseable
- Файловые потоки закрываются автоматически

### Логирование

- Важная практика для отладки и отслеживания ошибок
- Логи должны содержать достаточно информации для анализа проблемы
- Режим дозаписи (append mode) сохраняет все ошибки в одном файле
