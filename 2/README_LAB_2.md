# Лабораторная работа №2
## Объектно-ориентированное программирование на Java

---

## Описание

Данная лабораторная работа посвящена изучению и практическому применению основных концепций объектно-ориентированного программирования (ООП) в языке Java. В ходе работы реализована иерархия классов на тему компьютерной периферии, демонстрирующая механизмы наследования, инкапсуляции, полиморфизма и абстракции. Проект включает абстрактный базовый класс и три специализированных класса-наследника, каждый представляющий отдельный тип периферийного устройства.

---

## Ключевые концепции ООП в проекте

### Абстракция

Использование абстрактного класса `Peripheral` для определения общей структуры всех видов компьютерной периферии. Абстрактный класс содержит общие поля (название, цена, вес) и абстрактный метод `showInfo()`, который должен быть реализован в каждом подклассе. Это позволяет избежать создания экземпляров базового класса и гарантирует, что каждое устройство может вывести информацию о себе по-своему.

### Инкапсуляция

Применение модификаторов доступа для защиты данных объекта:
- **protected** — для полей, доступных подклассам и классам в одном пакете
- Использование методов доступа (геттеры) и установки значений (сеттеры) для контроля над изменением данных
- Пример: `getName()`, `setName(String name)`, `getPrice()`, `setPrice(int price)`

### Наследование

Создание двухуровневой иерархии классов:
- **Уровень 1**: Абстрактный класс `Peripheral` (суперкласс)
- **Уровень 2**: Три класса-наследника: `Keyboard`, `Headphones`, `GraphicsTablet`

Каждый подкласс использует ключевое слово `extends` для наследования функциональности родительского класса и добавляет собственные специфичные поля и методы.

### Полиморфизм

Реализация полиморфизма через два механизма:
- **Переопределение методов**: Каждый подкласс переопределяет метод `showInfo()` с использованием аннотации `@Override` для вывода информации в соответствии со своей структурой
- **Полиморфное поведение**: Использование массива типа `Peripheral[]` для хранения ссылок на объекты разных подклассов и вызова их переопределенных методов в цикле
- **Перегрузка методов**: Метод `applyDiscount()` в базовом классе может быть использован всеми подклассами

## Структура проекта

Проект организован в виде иерархии классов, представляющей систему компьютерной периферии:

### **Peripheral.java — Абстрактный базовый класс**

Определяет базовые характеристики для всех видов компьютерной периферии.

**Поля**:
- `name` (String) — название устройства
- `price` (int) — цена в условных единицах
- `weight` (int) — вес в граммах
- `count` (static int) — статический счётчик количества созданных объектов

**Конструкторы**:
- Параметризованный конструктор `Peripheral(String name, int price, int weight)` — инициализирует поля и увеличивает счётчик
- Конструктор по умолчанию `Peripheral()` — использует значения по умолчанию ("Unknown", 100, 500)

**Методы**:
- Геттеры: `getName()`, `getPrice()`, `getWeight()`, `getCount()` — получение значений полей
- Сеттеры: `setName()`, `setPrice()`, `setWeight()` — установка значений полей
- `abstract void showInfo()` — абстрактный метод для вывода информации
- `void applyDiscount(int percent)` — применение скидки к цене (перегружаемый метод)

---

### **Keyboard.java — Клавиатура**

Представляет компьютерную клавиатуру с дополнительными специфичными характеристиками.

**Специфичные поля**:
- `layout` (String) — раскладка клавиатуры (QWERTY, AZERTY и т.д.)
- `backlight` (boolean) — наличие подсветки
- `switches` (String) — тип переключателей (Mechanical, Membrane и т.д.)

**Конструкторы**:
- Параметризованный: принимает все поля, включая наследуемые от `Peripheral`
- По умолчанию: создаёт клавиатуру со стандартными значениями (QWERTY, подсветка включена, Cherry MX)

**Методы**:
- Геттеры: `getLayout()`, `isBacklight()`, `getSwitches()`
- Сеттеры: `setLayout()`, `setBacklight()`, `setSwitches()`
- `@Override showInfo()` — выводит информацию о клавиатуре с указанием типа

---

### **Headphones.java — Наушники**

Представляет наушники с характеристиками звука и типом подключения.

**Специфичные поля**:
- `type` (String) — тип наушников (Over-ear, In-ear, On-ear и т.д.)
- `connectivity` (int) — импеданс (сопротивление) в омах
- `frequency` (String) — диапазон воспроизводимых частот в герцах

**Конструкторы**:
- Параметризованный: инициализирует все поля
- По умолчанию: создаёт наушники со стандартными значениями (Over-ear, 32 Ом, 20-20000 Гц)

**Методы**:
- Геттеры: `getType()`, `getConnectivity()`, `getFrequency()`
- Сеттеры: `setType()`, `setConnectivity()`, `setFrequency()`
- `@Override showInfo()` — выводит информацию о наушниках с указанием характеристик

---

### **GraphicsTablet.java — Графический планшет**

Представляет планшет для рисования со специализированными параметрами.

**Специфичные поля**:
- `size` (String) — размер рабочей поверхности (A6, A5, A4 и т.д.)
- `pressureLevels` (int) — количество уровней давления пера
- `connectivity` (String) — тип подключения (USB, Bluetooth и т.д.)

**Конструкторы**:
- Параметризованный: инициализирует все поля с данными
- По умолчанию: создаёт планшет со стандартными значениями (A5, 2048 уровней давления, USB)

**Методы**:
- Геттеры: `getSize()`, `getPressureLevels()`, `getConnectivity()`
- Сеттеры: `setSize()`, `setPressureLevels()`, `setConnectivity()`
- `@Override showInfo()` — выводит информацию о планшете с технико-художественными характеристиками

---

### **Main.java — Главный класс**

Класс для демонстрации работы всех компонентов системы.

**Функциональность**:
- Создание объектов всех типов периферии с параметризованными конструкторами
- Демонстрация работы методов `showInfo()` для вывода информации о каждом устройстве
- Использование геттеров для получения свойств объектов
- Использование сеттеров для изменения свойств объектов
- Демонстрация полиморфизма через массив базового типа `Peripheral[]`
- Вывод статического счётчика количества созданных объектов

## Содержание заданий

### Задание 1: Создание иерархии классов

**Описание**: Разработана полная иерархия классов, состоящая из абстрактного базового класса и трёх специализированных подклассов, демонстрирующих все принципы ООП.

**Реализованная структура**:

```
        Peripheral (абстрактный класс)
             ↑
      ┌──────┼──────┐
      ↓      ↓      ↓
   Keyboard  Headphones  GraphicsTablet
```

**Основные компоненты**:

1. **Абстрактный класс** `Peripheral`
   - Содержит 3 общих поля: `name`, `price`, `weight`
   - Статическая переменная счётчика `count` для подсчёта количества созданных объектов
   - Конструкторы: параметризованный и по умолчанию
   - Геттеры и сеттеры для всех полей
   - Абстрактный метод `showInfo()`
   - Перегружаемый метод `applyDiscount(int percent)`

2. **Два уровня наследования**:
   - **Уровень 1**: Три класса наследуют прямо от `Peripheral`
   - Каждый класс содержит минимум 3 специфичных поля
   - Каждый класс содержит минимум 2 метода описания поведения (кроме геттеров/сеттеров)

3. **Примеры создания объектов**:
   ```java
   Keyboard k = new Keyboard("Gaming Keyboard", 120, 900,
                              "QWERTY", true, "Cherry MX");
   Headphones h = new Headphones("Bass Boost", 150, 250,
                                 "Closed-back", 64, "10-28000");
   GraphicsTablet t = new GraphicsTablet("Pro Art", 300, 800,
                                         "A5", 4096, "Bluetooth");
   ```

### Демонстрация принципов ООП

#### 1. **Абстракция**
```java
public abstract class Peripheral {
    // ...абстрактный метод
    public abstract void showInfo();
}
```
Абстрактный класс нельзя инстанцировать напрямую, он служит только шаблоном для подклассов.

#### 2. **Инкапсуляция**
```java
protected String name;      // Защищённое поле
private int price;          // Приватное поле (может быть)

public String getName() {   // Геттер
    return name;
}

public void setName(String name) {  // Сеттер
    this.name = name;
}
```

#### 3. **Наследование**
```java
public class Keyboard extends Peripheral {
    // Наследует все поля и методы от Peripheral
    private String layout;  // Новое специфичное поле
    
    public Keyboard(String name, int price, int weight,
                    String layout, boolean backlight, String switches) {
        super(name, price, weight);  // Вызов конструктора родителя
        this.layout = layout;
    }
}
```

#### 4. **Полиморфизм**

**Переопределение методов**:
```java
@Override
public void showInfo() {
    System.out.println("Клавиатура: " + name +
            ", раскладка: " + layout +
            ", подсветка: " + (backlight ? "есть" : "нет") +
            ", переключатели: " + switches +
            ", цена: " + price + " у.е., вес: " + weight + " г");
}
```

**Полиморфное поведение**:
```java
Peripheral[] devices = {k, h, t};
for (Peripheral p : devices) {
    p.showInfo();  // Вызывается переопределённый метод каждого подкласса
}
```

### Задание 3: Использование статического счётчика

**Описание**: Реализован механизм подсчёта созданных объектов с использованием статической переменной.

**Реализация**:
```java
public abstract class Peripheral {
    protected static int count = 0;  // Статическая переменная класса
    
    public Peripheral(String name, int price, int weight) {
        this.name = name;
        this.price = price;
        this.weight = weight;
        count++;  // Увеличение счётчика при создании объекта
    }
    
    public static int getCount() {
        return count;
    }
}
```

**Особенности**:
- Статическая переменная `count` принадлежит классу, а не отдельным объектам
- Общая для всех экземпляров классов из иерархии
- Увеличивается при вызове конструктора любого подкласса
- Получить значение можно через `Peripheral.getCount()`

## Важные замечания

- **Абстрактные классы**: Класс `Peripheral` не может быть инстанцирован напрямую, можно только создавать объекты его подклассов

- **Ключевое слово super**: Используется в конструкторах подклассов для вызова конструктора родительского класса и инициализации наследуемых полей

- **Аннотация @Override**: Используется при переопределении методов родительского класса. Помогает компилятору обнаружить ошибки и повышает читаемость кода

- **Модификаторы доступа**: 
  - `protected` — доступно подклассам и классам в одном пакете
  - `private` — доступно только внутри класса
  - отсутствие модификатора (package-private) — доступно в одном пакете

- **Статические члены класса**: Переменная `count` и метод `getCount()` принадлежат классу целиком, а не отдельным экземплярам

- **Полиморфизм через массив**: Массив типа `Peripheral[]` может содержать ссылки на объекты любых подклассов, что позволяет обрабатывать их единообразно
