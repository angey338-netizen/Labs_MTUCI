## Ключевые концепции

### Хэш-таблицы (Hash Tables)

Хэш-таблица — это структура данных, которая использует хэш-функцию для преобразования ключа в индекс массива, где хранится соответствующее значение. Это позволяет достичь очень быстрого доступа к элементам.

**Основной принцип**:
```
Ключ → [Хэш-функция] → Индекс массива → Значение
```

**Преимущества**:
- **Быстрый доиск**: В среднем O(1) — константное время доступа
- **Эффективность памяти**: Используется только необходимое количество место
- **Гибкость**: Можно использовать любые объекты в качестве ключей и значений
- **Масштабируемость**: Хорошо работает с большим количеством данных

**Недостатки**:
- **Коллизии**: Разные ключи могут получить один и тот же индекс
- **Сложность реализации**: Требует правильной хэш-функции
- **Неупорядоченность**: Элементы хранятся в неопределённом порядке

### Хэш-функция (Hash Function)

Функция, которая преобразует объект в целое число (хэш-код), а затем масштабирует его до диапазона индексов массива:

```java
private int hash(K key) {
    return Math.abs(key.hashCode()) % capacity;
}
```

**Свойства хорошей хэш-функции**:
- **Быстрота**: Вычисляется за константное время
- **Распределение**: Равномерно распределяет ключи по всему массиву
- **Детерминированность**: Одинаковый ключ всегда даёт одинаковый результат
- **Минимизация коллизий**: Максимально избегает совпадений

### Коллизии и их разрешение

**Коллизия** — это ситуация, когда двум разным ключам соответствует одинаковый индекс в массиве.

**Пример коллизии**:
```
hash("apple") = 5
hash("pineapple") = 5  // Коллизия!
```

**Методы разрешения коллизий**:

1. **Метод цепочек (Chaining)** — используется в данной работе:
   - Каждая ячейка массива содержит список всех пар ключ-значение
   - При коллизии новая пара добавляется в список
   - Простая в реализации, но может потребовать доп. память

2. **Открытая адресация (Open Addressing)**:
   - При коллизии ищется другая свободная ячейка
   - Может быть линейная (linear probing) или квадратичная (quadratic)
   - Использует меньше памяти, но сложнее в реализации

**Визуализация метода цепочек**:
```
Исходная таблица:
Index 0: 
Index 1: 
Index 2: ("banana", 3)
Index 3: ("orange", 7)
Index 4: 
Index 5: ("apple", 5)

После добавления ("pear", 2) с коллизией на индексе 2:
Index 0: 
Index 1: 
Index 2: ("banana", 3) → ("pear", 2)
Index 3: ("orange", 7)
Index 4: 
Index 5: ("apple", 5)
```

### Обобщённые типы (Generics)

Параметризованные типы, позволяющие создавать классы и методы, работающие с различными типами данных при гарантии типобезопасности на этапе компиляции.

**Синтаксис**:
```java
public class HashTable<K, V> {
    // K - параметр типа для ключа (Key)
    // V - параметр типа для значения (Value)
}
```

**Преимущества дженериков**:
- **Типобезопасность**: Компилятор проверяет типы, ошибки выявляются раньше
- **Отсутствие приведения типов**: Не нужно писать (Type) cast
- **Повторное использование кода**: Один класс работает с разными типами
- **Читаемость**: Явное указание типов улучшает понимание кода

**Примеры использования**:
```java
HashTable<String, Integer> numbers = new HashTable<>(16);
HashTable<String, Product> products = new HashTable<>(32);
HashTable<Long, Customer> customers = new HashTable<>(64);
```

### Сравнение: собственная реализация vs HashMap

| Аспект | Собственная HashTable | Встроенная HashMap |
|--------|---------------------|-------------------|
| **Понимание** | Обучает принципам | Использует готовый API |
| **Реализация** | Явное управление | Оптимизированная реализация |
| **Производительность** | O(1) в среднем | O(1) оптимизировано |
| **Расширение** | Вручную задать ёмкость | Автоматическое расширение |
| **Функциональность** | Минимальная | Полнофункциональная |
| **Надёжность** | Образовательная | Production-ready |

## Структура проекта

### **HashTable.java — Собственная реализация хэш-таблицы**

Типобезопасная реализация хэш-таблицы с использованием дженериков, демонстрирующая принципы работы структуры данных.

**Параметры типа**:
- `<K, V>` — ключ и значение любого типа

**Внутренняя структура**:

```java
private final LinkedList<Entry<K, V>>[] table;  // Массив связных списков
private final int capacity;                      // Размер массива (количество "бакетов")
private int size;                                // Количество добавленных элементов
```

**Вложенный класс Entry<K, V>**:
```java
private static class Entry<K, V> {
    private K key;    // Ключ для поиска
    private V value;  // Хранимое значение
}
```
Представляет одну пару ключ-значение. Используется вместе с LinkedList для организации цепочек при коллизиях.

**Конструктор**:
```java
public HashTable(int capacity) {
    this.capacity = capacity;
    this.size = 0;
    this.table = new LinkedList[capacity];
    
    for (int i = 0; i < capacity; i++) {
        table[i] = new LinkedList<Entry<K,V>>();
    }
}
```
- Инициализирует массив размером `capacity`
- Каждая ячейка содержит пустой LinkedList
- LinkedList будет расширяться при возникновении коллизий

**Метод hash(K key)** — вычисление индекса:
```java
private int hash(K key) {
    return Math.abs(key.hashCode()) % capacity;
}
```
- Использует встроенный метод `hashCode()` объекта
- Берёт абсолютное значение для избежания отрицательных индексов
- Масштабирует результат до диапазона [0, capacity)

**Метод put(K key, V value)** — добавление/обновление:

Алгоритм:
1. Вычислить индекс по ключу
2. Получить цепочку по этому индексу
3. Проверить, есть ли уже такой ключ в цепочке
4. Если найден — обновить значение и вернуться
5. Если не найден — добавить новую пару в конец цепочки
6. Увеличить счётчик размера

```java
public void put(K key, V value) {
    int index = hash(key);
    
    // Проверка на существование ключа
    for (Entry<K, V> entry : table[index]) {
        if (entry.key.equals(key)) {
            entry.value = value;  // Обновление
            return;
        }
    }
    
    // Добавление новой пары
    table[index].add(new Entry<>(key, value));
    size++;
}
```

**Метод get(K key)** — получение значения:

Алгоритм:
1. Вычислить индекс
2. Обойти цепочку по этому индексу
3. Найти запись с нужным ключом
4. Вернуть значение или null, если не найдено

```java
public V get(K key) {
    int index = hash(key);
    
    for (Entry<K, V> entry : table[index]) {
        if (entry.key.equals(key)) {
            return entry.value;
        }
    }
    
    return null;
}
```

**Метод remove(K key)** — удаление:

Алгоритм:
1. Найти цепочку по индексу ключа
2. Найти запись с нужным ключом
3. Удалить запись из цепочки
4. Уменьшить счётчик размера
5. Вернуть удалённое значение

```java
public V remove(K key) {
    int index = hash(key);
    
    for (int i = 0; i < table[index].size(); i++) {
        Entry<K, V> entry = table[index].get(i);
        if (entry.key.equals(key)) {
            V oldValue = entry.value;
            table[index].remove(i);
            size--;
            return oldValue;
        }
    }
    
    return null;
}
```

**Вспомогательные методы**:
- `size()` — возвращает количество элементов в таблице
- `isEmpty()` — проверяет, пуста ли таблица

## Важные замечания

### Дженерики и типобезопасность

```java
HashTable<String, Integer> table = new HashTable<>(16);
table.put("apple", 5);
Integer value = table.get("apple");  // Без необходимости приведения типов
```

Компилятор гарантирует, что переменная `value` имеет тип Integer и не требует явного приведения.

### Хэш-функция и коллизии

```java
private int hash(K key) {
    return Math.abs(key.hashCode()) % capacity;
}
```

- `Math.abs()` убедитет, что индекс всегда положительный
- `% capacity` масштабирует результат до допустимого диапазона
- Коллизии обрабатываются методом цепочек (LinkedList)

### Метод equals() для сравнения

```java
if (entry.key.equals(key)) {
    // Нашли нужный ключ
}
```

Используется метод `equals()`, а не `==`, потому что нужно сравнивать содержимое, а не ссылки на объекты.

### HashMap vs собственная HashTable

- **HashMap**: Встроенная реализация, оптимизированная, с автоматическим расширением
- **Собственная HashTable**: Образовательная цель — понимание внутренних механизмов

В реальных приложениях используется HashMap, но понимание принципов работы критично для эффективной разработки.

### Валидация данных

```java
if (barcode == null || barcode.isBlank()) {
    throw new IllegalArgumentException("Штрихкод не может быть пустым");
}
```

Критично для корректной работы хэширования, так как null или пустая строка могут вызвать ошибки при вычислении хэша.
