## Ключевые концепции

### Хэш-таблицы (Hash Tables)

Хэш-таблица — это структура данных, которая использует хэш-функцию для преобразования ключа в индекс массива, где хранится значение. Основные преимущества:
- **Быстрый поиск**: В среднем операции выполняются за O(1)
- **Эффективность памяти**: Используется только необходимое количество места
- **Универсальность**: Можно использовать любые объекты в качестве ключей

### Хэш-функция (Hash Function)

Преобразует ключ в индекс массива:
```java
private int hash(K key) {
    return Math.abs(key.hashCode()) % capacity;
}
```

Свойства хорошей хэш-функции:
- Быстрота вычисления
- Равномерное распределение значений
- Детерминированность (одинаковый ключ всегда даёт одинаковый результат)

### Коллизии (Collisions)

Ситуация, когда двум разным ключам соответствует одна позиция в таблице. В данной реализации используется **метод цепочек** (chaining), где каждая позиция содержит связный список (LinkedList) всех пар ключ-значение с одинаковым хэшем.

**Пример коллизии**:
```
Index 0: 
Index 1: 
Index 2: ("banana", 3) → ("pear", 2)     // Коллизия: два элемента на одной позиции
Index 3: ("orange", 7)
Index 4: 
Index 5: ("apple", 5)
```

### Обобщённые типы (Generics)

Параметризованные типы данных, позволяющие создавать классы и методы, работающие с различными типами:
- **Type Safety**: Проверка типов на этапе компиляции
- **Code Reusability**: Один класс работает с разными типами
- **Elimination of Casting**: Не нужны приведения типов

**Синтаксис**:
```java
public class HashTable<K, V> {  // K - ключ, V - значение
    // Код класса
}
```

### Методы хэш-таблицы

- **put(K key, V value)**: Добавляет или обновляет пару ключ-значение
- **get(K key)**: Получает значение по ключу
- **remove(K key)**: Удаляет пару по ключу и возвращает значение
- **size()**: Возвращает количество элементов
- **isEmpty()**: Проверяет, пуста ли таблица

## Структура проекта

### **HashTable.java — Собственная реализация хэш-таблицы**

Обобщённый класс, реализующий хэш-таблицу с методом цепочек.

**Параметры типа**:
- `K` — тип ключей
- `V` — тип значений

**Вложенный класс Entry**:
```java
private static class Entry<K, V> {
    private K key;      // Ключ
    private V value;    // Значение
}
```
Представляет пару ключ-значение, хранящуюся в таблице.

**Поля класса**:
- `table` — массив связных списков (LinkedList<Entry<K, V>>[])
- `capacity` — размер массива (количество "бакетов")
- `size` — количество элементов в таблице

**Конструктор**:
```java
public HashTable(int capacity) {
    this.capacity = capacity;
    this.size = 0;
    this.table = new LinkedList[capacity];
    for (int i = 0; i < capacity; i++) {
        table[i] = new LinkedList<Entry<K,V>>();
    }
}
```
Инициализирует массив связных списков размером `capacity`.

**Метод hash()**:
```java
private int hash(K key) {
    return Math.abs(key.hashCode()) % capacity;
}
```
Вычисляет индекс позиции для ключа.

**Метод put(K key, V value)**:
- Вычисляет хэш ключа
- Если ключ уже существует, обновляет значение
- Если ключа нет, добавляет новую пару в конец цепочки
- Увеличивает счётчик размера

**Метод get(K key)**:
- Вычисляет хэш ключа
- Находит цепочку по индексу
- Ищет пару с указанным ключом в цепочке
- Возвращает значение или null

**Метод remove(K key)**:
- Находит и удаляет пару с указанным ключом
- Возвращает удалённое значение
- Уменьшает счётчик размера

## Содержание заданий

### Задание 1: Собственная реализация HashTable

**Описание**: Разработана полностью функциональная хэш-таблица, реализующая метод цепочек для разрешения коллизий.

**Реализованные функции**:

1. **Класс Entry<K, V>**:
   - Вложенный класс для хранения пар ключ-значение
   - Содержит два поля: key и value
   - Используется внутри LinkedList в каждой цепочке

2. **Инициализация таблицы**:
```java
HashTable<String, Integer> table = new HashTable<>(16);
```
Создаёт таблицу ёмкостью 16 "бакетов" (позиций массива).

3. **Операция put() — Добавление/обновление**:
```java
table.put("apple", 5);
table.put("banana", 3);
table.put("orange", 7);
table.put("banana", 10);  // Обновление
```
- Если ключ существует, значение обновляется
- Если ключа нет, добавляется новая пара
- Коллизии разрешаются добавлением в конец цепочки

4. **Операция get() — Получение значения**:
```java
Integer value = table.get("banana");  // Возвращает 3, затем 10
```
- Поиск по цепочке при совпадении хэша
- Возвращает null, если ключ не найден

5. **Операция remove() — Удаление**:
```java
Integer removed = table.remove("apple");  // Возвращает 5
```
- Удаляет пару из таблицы
- Возвращает удалённое значение
- Уменьшает счётчик размера

6. **Вспомогательные методы**:
```java
table.size();      // Возвращает количество элементов
table.isEmpty();   // Проверяет пустоту
```

**Пример вывода**:
```
Размер таблицы: 3
Значение по ключу banana: 3
Обновленное значение banana: 10
Удалено значение по ключу apple: 5
Размер таблицы после удаления: 2
Пуста ли таблица? false
```

**Алгоритм put()**:
```java
public void put(K key, V value) {
    int index = hash(key);                    // 1. Вычислить индекс
    
    // 2. Проверить, есть ли уже такой ключ
    for (Entry<K, V> entry : table[index]) {
        if (entry.key.equals(key)) {
            entry.value = value;              // Обновить значение
            return;
        }
    }
    
    // 3. Если ключа нет, добавить новую пару
    table[index].add(new Entry<>(key, value));
    size++;
}
```

## Важные замечания

### Обобщённые типы (Generics)
- **Тип безопасность**: Компилятор проверяет типы на этапе компиляции
- **Нет требуемых приведений**: Можно сразу использовать значение нужного типа
- **Параметры типов**: `<K, V>` — буквы соглашения для Ключа (Key) и Значения (Value)

### Хэширование
- **Хэш-функция**: `Math.abs(key.hashCode()) % capacity` преобразует объект в индекс
- **Коллизии неизбежны**: Метод цепочек позволяет хранить несколько элементов на одной позиции
- **Загрузочный фактор**: Отношение количества элементов к размеру таблицы влияет на производительность

### HashMap vs собственная HashTable
- **HashMap**: Встроенная реализация с автоматическим расширением
- **Собственная HashTable**: Образовательная цель — понимание внутренних механизмов

### Проверка входных данных
```java
if (barcode == null || barcode.isBlank()) {
    throw new IllegalArgumentException("Штрихкод не может быть пустым");
}
```
Критично для корректной работы хэш-таблицы.

### forEach в Java
```java
products.forEach((barcode, product) -> {
    System.out.println("Штрихкод: " + barcode + " | " + product);
});
```
Удобный способ итерации по HashMap (требует Java 8+).
