# Лабораторная работа №7
## Многопоточность в Java

---

## Описание

Данная лабораторная работа посвящена изучению механизмов многопоточного программирования в Java. В ходе работы реализованы три программы: параллельное вычисление суммы элементов массива с разделением на два потока, многопоточный поиск максимального элемента в матрице с отдельным потоком на каждую строку, и симуляция работы склада с тремя грузчиками, синхронизированными через `CyclicBarrier`. Работа направлена на освоение создания потоков, управления их жизненным циклом и применения механизмов синхронизации из пакета `java.util.concurrent`.

---

## Ключевые концепции

### Поток (Thread)

Поток — это наименьшая единица выполнения программы. Java позволяет запускать несколько потоков одновременно в рамках одного процесса, что даёт возможность параллельно обрабатывать данные и сокращать общее время выполнения.

Создание потока через лямбда-выражение (реализация `Runnable`):
```java
Thread thread = new Thread(() -> {
    // код, выполняемый в отдельном потоке
});
thread.start();
```

Создание потока через класс, реализующий `Runnable`:
```java
static class Loader implements Runnable {
    @Override
    public void run() {
        // код потока
    }
}

Thread t = new Thread(new Loader(), "Грузчик 1");
t.start();
```

### Метод join()

`join()` заставляет текущий поток ждать завершения другого потока. Используется в главном потоке (`main`), чтобы не продолжать выполнение до тех пор, пока все рабочие потоки не закончат свою работу:

```java
thread1.start();
thread2.start();

thread1.join(); // main ждёт завершения thread1
thread2.join(); // main ждёт завершения thread2

// только после этого выполняется следующая строка
System.out.println("Все потоки завершены.");
```

### Статические поля как общая память

Потоки одного процесса разделяют общую память. Статические поля класса доступны всем потокам напрямую, что позволяет передавать результаты вычислений из рабочих потоков в главный:

```java
static int sum1 = 0;
static int sum2 = 0;

// поток 1 записывает в sum1, поток 2 — в sum2
// главный поток читает оба значения после join()
```

### final-переменные в лямбдах

Лямбда-выражения в Java могут обращаться только к переменным, которые являются `final` или фактически неизменными (effectively final). При использовании переменной цикла внутри лямбды необходимо скопировать её в отдельную финальную переменную:

```java
for (int row = 0; row < matrix.length; row++) {
    final int r = row; // копия для захвата лямбдой
    threads[row] = new Thread(() -> {
        // используем r, а не row
        int max = matrix[r][0];
    });
}
```

### CyclicBarrier

`CyclicBarrier` — барьер синхронизации, который блокирует каждый поток в точке `await()` до тех пор, пока все участники (заданное количество потоков) не достигнут этой точки. После этого барьер открывается, опционально выполняется барьерное действие, и все потоки продолжают работу:

```java
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    // барьерное действие — выполняется один раз, когда все 3 потока вызвали await()
    System.out.println("Все грузчики собрались. Едем!");
});

// в каждом потоке:
barrier.await(); // поток блокируется и ждёт остальных
```

Слово «Cyclic» означает, что барьер можно использовать повторно: после срабатывания он автоматически сбрасывается и готов к следующему циклу.

### Исключения в многопоточности

- **`InterruptedException`** — выбрасывается, когда поток прерывается во время ожидания (`join()`, `await()`). При перехвате необходимо восстановить флаг прерывания: `Thread.currentThread().interrupt()`.
- **`BrokenBarrierException`** — выбрасывается методом `await()`, если барьер был нарушен (например, один из ожидающих потоков был прерван).

## Содержание заданий

### Задание 1: Параллельное вычисление суммы массива (Вариант 1) — `SimpleTask1ArraySum`

**Описание**: Массив из 10 элементов делится на две половины. Каждая половина суммируется в отдельном потоке. После завершения обоих потоков главный поток складывает частичные суммы и выводит итог.

**Исходный массив**:
```java
int[] numbers = {8, 4, 1, 9, 3, 5, 8, 6, 10, 11};
// Первая половина:  {8, 4, 1, 9, 3}  → sum1 = 25
// Вторая половина: {5, 8, 6, 10, 11} → sum2 = 40
```

**Разделение массива между потоками**:
```java
// Поток 1 — индексы от 0 до numbers.length / 2 (не включая)
Thread thread1 = new Thread(() -> {
    for (int i = 0; i < numbers.length / 2; i++) {
        sum1 += numbers[i];
    }
});

// Поток 2 — индексы от numbers.length / 2 до конца
Thread thread2 = new Thread(() -> {
    for (int i = numbers.length / 2; i < numbers.length; i++) {
        sum2 += numbers[i];
    }
});
```

**Запуск и ожидание завершения**:
```java
thread1.start();
thread2.start();

thread1.join(); // main ждёт завершения потока 1
thread2.join(); // main ждёт завершения потока 2

System.out.println("Общая сумма: " + (sum1 + sum2));
```

Без `join()` главный поток мог бы прочитать `sum1` и `sum2` до того, как рабочие потоки завершили вычисления, и получить некорректный результат.

**Пример вывода**:
```
Сумма первой половины: 25
Сумма второй половины: 40
Общая сумма: 65
```

### Задание 2: Поиск максимального элемента в матрице (Вариант 1) — `SimpleTask2MatrixMax`

**Описание**: Для каждой строки матрицы создаётся отдельный поток, который находит максимальный элемент в своей строке и записывает результат в общий массив `rowMax`. После завершения всех потоков главный поток находит глобальный максимум среди локальных максимумов строк.

**Исходная матрица**:
```
 5  14   2   8   → max строки = 14
19   3   7  11   → max строки = 19
 6  25   1   4   → max строки = 25
13  10  17   9   → max строки = 17
```

**Создание потока для каждой строки**:
```java
static int[] rowMax; // массив локальных максимумов

for (int row = 0; row < matrix.length; row++) {
    final int r = row; // final-копия для использования в лямбде

    threads[row] = new Thread(() -> {
        int max = matrix[r][0]; // начальное значение — первый элемент строки
        for (int col = 0; col < matrix[r].length; col++) {
            if (matrix[r][col] > max) {
                max = matrix[r][col];
            }
        }
        rowMax[r] = max; // запись результата в общий массив
    });

    threads[row].start();
}
```

**Ожидание всех потоков и поиск глобального максимума**:
```java
for (int i = 0; i < threads.length; i++) {
    threads[i].join();
}

int answer = rowMax[0];
for (int i = 0; i < rowMax.length; i++) {
    if (rowMax[i] > answer) {
        answer = rowMax[i];
    }
}

System.out.println("Наибольший элемент матрицы: " + answer);
```

Каждый поток записывает в **свою** ячейку массива `rowMax[r]`, поэтому конкурентный доступ к одному индексу отсутствует и синхронизация не требуется.

**Пример вывода**:
```
Наибольший элемент матрицы: 25
```

### Задание 3: Симуляция работы склада (Вариант 9) — `SimpleTask3WarehouseCyclicBarrier`

**Описание**: Три грузчика работают параллельно. Каждый берёт товар заданного веса и ждёт остальных у барьера. Как только все трое собрались, `CyclicBarrier` срабатывает: выполняется барьерное действие (подсчёт веса и отправка на другой склад), после чего грузчики начинают следующую ходку. Процесс повторяется три раза.

**Данные о ходках**:
```java
static int[][] trips = {
    {50, 40, 60}, // ходка 1: грузчик1=50кг, грузчик2=40кг, грузчик3=60кг → итого 150кг
    {70, 30, 50}, // ходка 2: 70кг + 30кг + 50кг → итого 150кг
    {40, 50, 60}  // ходка 3: 40кг + 50кг + 60кг → итого 150кг
};
```

**Настройка барьера**:
```java
static CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    // барьерное действие: выполняется когда все 3 потока вызвали await()
    int totalWeight = trips[currentTrip][0]
                    + trips[currentTrip][1]
                    + trips[currentTrip][2];

    System.out.println("Собрано " + totalWeight + " кг. Грузчики едут на другой склад.");
    System.out.println("Товары разгружены. Грузчики вернулись.\n");

    currentTrip++; // переходим к следующей ходке
});
```

- Первый аргумент `3` — количество потоков, которые должны вызвать `await()` для срабатывания барьера.
- Второй аргумент — `Runnable`, выполняемый один раз при каждом срабатывании барьера, до того как потоки продолжат работу.

**Класс `Loader` — логика грузчика**:
```java
static class Loader implements Runnable {
    int loaderNumber;

    Loader(int loaderNumber) {
        this.loaderNumber = loaderNumber;
    }

    @Override
    public void run() {
        try {
            for (int trip = 0; trip < trips.length; trip++) {
                int weight = trips[trip][loaderNumber];

                System.out.println(Thread.currentThread().getName()
                        + " взял товар весом " + weight + " кг");

                System.out.println(Thread.currentThread().getName()
                        + " ждёт остальных грузчиков");

                barrier.await(); // ждать всех у барьера перед отправкой
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } catch (BrokenBarrierException e) {
            System.out.println("Барьер был нарушен.");
        }
    }
}
```

**Запуск потоков**:
```java
Thread loader1 = new Thread(new Loader(0), "Грузчик 1");
Thread loader2 = new Thread(new Loader(1), "Грузчик 2");
Thread loader3 = new Thread(new Loader(2), "Грузчик 3");

loader1.start();
loader2.start();
loader3.start();

loader1.join();
loader2.join();
loader3.join();

System.out.println("Работа закончена. Все товары перенесены.");
```

**Схема работы барьера (одна ходка)**:
```
Грузчик 1: взял 50 кг → await() → блокирован
Грузчик 2: взял 40 кг → await() → блокирован
Грузчик 3: взял 60 кг → await() → ВСЕ СОБРАЛИСЬ
                                  → барьерное действие: 150 кг, едем!
                                  → все три потока продолжают
```

**Пример вывода** (порядок строк от потоков может варьироваться):
```
Грузчик 1 взял товар весом 50 кг
Грузчик 1 ждёт остальных грузчиков
Грузчик 2 взял товар весом 40 кг
Грузчик 2 ждёт остальных грузчиков
Грузчик 3 взял товар весом 60 кг

Грузчик 2 ждёт остальных грузчиков
```

## Важные замечания

### Недетерминированность порядка вывода

Операционная система сама решает, какой поток выполнять в каждый момент времени. Порядок строк в выводе заданий 2 и 3 может отличаться при каждом запуске программы — это нормальное поведение многопоточной программы. Результат (сумма, максимум, итог склада) при этом остаётся корректным.

### Зачем нужна final-копия переменной цикла

Переменная `row` в задании 2 изменяется на каждой итерации цикла. Если бы лямбда захватывала её напрямую, к моменту выполнения потока значение `row` уже могло измениться. Создание `final int r = row` фиксирует текущее значение для каждого потока отдельно.

### CyclicBarrier vs CountDownLatch

`CountDownLatch` — одноразовый: после срабатывания не сбрасывается. `CyclicBarrier` — многоразовый: после каждого срабатывания автоматически сбрасывается и готов к следующему циклу. Именно поэтому в задании 3 один барьер используется для трёх последовательных ходок.

### Отсутствие синхронизации в задании 1

В задании 1 каждый поток записывает в **свою** переменную (`sum1` или `sum2`), поэтому конкурентного доступа к одним данным нет и явная синхронизация не требуется. Однако если оба потока писали бы в одну переменную, это привело бы к состоянию гонки и некорректному результату.

### Порядок исполнения барьерного действия

Барьерное действие (второй аргумент `CyclicBarrier`) выполняется в потоке, который последним вызвал `await()`, до того как барьер откроется для всех остальных. Это гарантирует, что `currentTrip++` произойдёт до начала следующей ходки.

### Восстановление флага прерывания

При перехвате `InterruptedException` важно вызывать `Thread.currentThread().interrupt()`. Это восстанавливает флаг прерывания потока, который был сброшен при выбрасывании исключения, и позволяет вышестоящему коду корректно отреагировать на прерывание.
